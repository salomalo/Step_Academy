Базові поняття
XHTML – це нова версія широко використовуваної мови гіпертекстової розмітки HTML (Hypertext Markup Language), яка побудована в відповідності до правил розширюваної мови розмітки XML (Extensible Markup Language). Фактично XHTML успадковує простоту, а також синтаксис і розмітку HTML та примушує його функціонувати по правилам XML при описі та обробці розмітки.
Сама мова HTML була розроблена на основі мови створення мов розмітки SGML (Standard Generalized Markup Language), яка стала стандартом в 1969 р. завдяки її розробнику Чарльзу Гольдфарбу, що працював в IBM. Першими версіями мови HTML стали HTML 1.0, HTML 1.2 та HTML 2.0, які теж розробляв Тім Бернард Лі. Доречі, стандарт HTML 2.0 - вже був випущений під егідою Консорціумом World Wide Web (W3C), яка на сьогоднішній день являється міжнародним некомерційним об’єднанням, до якого входять більше 150 фірми-розробники браузерів, незалежні експерти та комерційні структури.
Наступними версіями стали HTML 3.0, HTML 3.2 і 4, які вже включали підтримку каскадних таблиць стилів CSS та  суттєвої різниці між якими не було. Механізм CSS каскадних таблиць стилів використовувався для того, щоб відокремити структуру документу від його форматування. На сьогоднішній день існує версія CSS 2.0.
В результаті всіх видозмін була випущена XHTML 1.1- мова, яка використовує елементи HTML записані за правилами XML-мови, що містить правила та можливість створення власних елементів розмітки web-сторінки. Отже, тепере якщо творця сторінки не задовольняють стандартні елементи XHTML він може створити свої власні.

Згідно стандартів XHTML 1.1, весь вміст повинен кодуватись всередині блочних тегів, щоб розрізняти окремі блоки, які розміщуються на сторінці. Всі лінійні теги повинні розміщуватись всередині блочних.

Для того, щоб текстовий файл став HTML-файлом, змінити його розширення з ".txt" на ".html" недостатньо. Потрібно дотримуватись «правило першої відповідності»: кожен HTML-документ, який відповідає специфікації HTML довільної версії, повинен обов’язково починатись з рядка декларації версії HTML !DOCTYPE, яка зазвичай виглядає наступним чином:
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.0 Transitional//EN">
<!DOCTYPE HTML PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN">
Цей рядок допоможе браузеру визначити як вірно інтерпретувати даний документ. 
Правила запису тегів та атрибутів в XHTML
Всі теги пишуться в нижньому регістрі.
Контейнерні теги повинні мати відкриваючий та закриваючий теги. 
Крім того, сторінка повинна містити теги документа <html></html>.
Кирилиця при написанні тегів не допускається.
Між оператором ( < ) та назвою тега не повинно бути пропуска (space).
Одинарні (лінійні) теги не потребують закриваючого тега, але повинні закінчуватись символом ( / ). 
Вкладеність тегів в XML, і як наслідок в XHTML строго контролюється. Тобто в HTML можна зробити наступний “ізврат”:<a>La-la-la <b> Tru-la-la </a> He-he-he </b>

Теги можуть мати атрибути, які задають додаткові характеристики для елемента. Атрибути складаються з пар «атрибут=”значення”». Причому значення атрибута задається в подвійних або одинарних лапких. Наприклад, ми можемо змінити колір нашої Web-сторінки.
<!DOCTYPE HTML PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN">
<html>
<head>
     <title>Мій перший крок </title>
</head>

<body bgcolor="yellow" text="#cc0000">
    Добрий день. Це моя перша сторінка.
</body>
</html>
Крім того, якщо значенням є колір, то він може записуватись словом або шістнадцятковим представленням. В останньому випадку перед значенням вказується символ # (хеш).
Існує ряд застарівших та небажаних атрибутів, замість яких рекомендується використовувати каскадні таблиці стилів CSS. До таких атрибутів відносять align (для вирівнювання тексту), bgcolor (колір фону), valign (вертикальне вирівнювання), text (колір тексту документа) тощо. По ходу нашого знайомства з мовою розмітки ми з такими атрибутами ще познайомимось.
